# OpenClaw 文件系统治理标准 (FS Governance)

本标准旨在解决 OpenClaw 根目录 (`~/.openclaw/`) 随时间推移导致的“扁平化爆炸”和文件管理失控问题，确保系统长期运行下的可维护性与安全性。

## 1. 核心分区协议 (Partition Protocol)

严禁在根目录新建任何非协议文件夹。所有文件必须归档至以下五大逻辑分区：

| 分区 | 用途 (Purpose) | 关键特性 |
| :--- | :--- | :--- |
| **`etc/`** | 持久配置 | 系统灵魂，包含 `config.json` 及关键安全补丁。**禁止清理。** |
| **`var/`** | 运行时状态 | 系统脉搏，包含 `logs/`, `memory/`, `cron/` 状态数据。**需定期归档。** |
| **`lib/`** | 资产库 | 系统能力定义，包含 `skills/`, `agents/`。**禁止清理。** |
| **`work/`** | 成果区 | 用户开发产出，如 `workspace/`。**按需维护。** |
| **`cache/`** | 临时/缓存 | 系统消耗品，包含 `browser/`, `tmp/`。**可随时全自动清理。** |
| **`bin/`** | 工具链 | 维护脚本与自动化工具，包含 `maintenance/`。**禁止清理。** |

## 2. 自动化治理规范 (Governance Loop)

- **唯一性原则**：代码与配置严禁冗余备份。使用符号链接 (Symlinks) 桥接旧路径与新结构。
- **自动化防抖**：所有清理脚本必须包含“重启冷却锁” (Lockfile)，防止监控任务引发的循环重启。
- **防御性检查**：在执行任何 Git 提交前，必须运行 `scripts/pre-commit-check.sh`，检查是否存在敏感信息泄露或违规文件创建。

## 3. 运维操作手册

### 自动清理：
- 执行根目录零散文件清理：`./scripts/cleanup-root.sh`
- 执行智能垃圾回收 (GC)：`./scripts/auto-gc.sh` (需配置)

### 新文件创建规则：
1. 先确认目标分区 (`etc/`, `var/`, `lib/` 等)。
2. 若需在根目录新增路径，必须先更新此标准文档并获得确认。
3. 任何临时性质的脚本或测试文件，严禁存放在根目录，统一存放在 `var/archive/`。

## 4. 防御性治理逻辑
- 任何位于根目录且非核心分区的文件，默认被视为“待归档对象”。
- 脚本检测到根目录出现非协议路径时，应发出警报。
